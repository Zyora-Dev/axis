Axis vs. PyTorch: 7B-Parameter LoRA Fine-Tuning Performance

A controlled single-GPU comparison of training throughput and memory footprint

§Executive summary

On a single NVIDIA A100-80GB, Axis performs low-rank (LoRA) fine-tuning of a 7-billion-parameter transformer in 17% less GPU memory than the reference PyTorch stack (Hugging Face Transformers + PEFT), while PyTorch retains a ~15% throughput advantage. Both frameworks train an identical set of ~40M adapter parameters under matched precision and attention settings.

1Objective

This report quantifies the training-time throughput and peak device-memory consumption of the Axis compiled CUDA engine relative to the mainstream PyTorch fine-tuning stack, at a representative 7B model scale. Full-parameter 7B training does not fit within a single 80 GB accelerator for either framework (bf16 weights + gradients + fp32 Adam state ≈ 112 GB); the parameter-efficient LoRA regime is therefore the appropriate single-GPU workload and is the subject of this comparison.

4Methodology

  • Matched configuration. Both frameworks train the identical LoRA adapter set (same rank and target projections), the same model shape, and the same bf16 + flash-attention path.
  • Measurement. 2 warm-up steps precede 5 timed steps; wall-clock time is captured with device synchronisation around the timed region. Reported latency is the per-step mean.
  • Throughput. Tokens per second = (batch × sequence length) ÷ mean step time.
  • Memory. Peak device memory is read from the driver (nvidia-smi, memory-used) and reflects the full process footprint.
  • Isolation. Each configuration executes in its own fresh container to eliminate allocator and cache carry-over.

6Analysis

Memory. Axis's advantage stems from the parameter-efficient design of its compiled engine: the frozen 7B base carries no optimizer state and is held in bf16, so only the ~40M adapters incur fp32 master/moment storage. This yields a 4.9 GB (17%) reduction in peak footprint, translating directly into headroom for longer context windows or larger micro-batches on the same accelerator.

Throughput. PyTorch remains ~15% faster per step, reflecting a mature stack of hand-tuned CUDA kernels and deep PEFT integration. Axis closes much of this gap through CUDA-graph replay and fused flash attention but does not yet lead on raw LoRA throughput at this scale.

Consistency. The result is directionally consistent with the corresponding 1B-scale measurement (Axis ~38% lower LoRA memory, PyTorch marginally faster), indicating the memory advantage is structural rather than scale-specific.

7Scope and limitations

  • Results characterise LoRA fine-tuning only. Full-parameter 7B training exceeds a single 80 GB accelerator for both frameworks and requires multi-GPU sharding (ZeRO / FSDP), evaluated separately.
  • Measurements use a fixed sequence length (2,048) and batch size (1); throughput scales with both and should be re-measured for other regimes.
  • Reference-stack library versions track the current public release at the report date and are not pinned.

8Reproducibility

The benchmark harness and this report are published in the Axis repository. To reproduce on an A100-80GB:
